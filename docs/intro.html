简介¶

This is the documentation for the Sphinx documentation builder. Sphinx is a tool that translates a set of reStructuredText source files into various output formats, automatically producing cross-references, indices, etc. That is, if you have a directory containing a bunch of reST-formatted documents (and possibly subdirectories of docs in there as well), Sphinx can generate a nicely-organized arrangement of HTML files (in some other directory) for easy browsing and navigation. But from the same source, it can also generate a PDF file using LaTeX, rinohtype or rst2pdf (see 可用的构建器).

The focus is on hand-written documentation, rather than auto-generated API docs. Though there is support for that kind of documentation as well (which is intended to be freely mixed with hand-written content), if you need pure API docs have a look at Epydoc, which also understands reST.

For a great “introduction” to writing docs in general – the whys and hows, see also Write the docs, written by Eric Holscher.

从其他文档格式导入¶

这个部分列举转换文档到 reStructuredText/Sphinx 的小技巧。

  • Gerard Flanagan 写了一个脚本可以把 HTML 文件转换为 reST，参见 Python Package Index。
  • 如果要把旧的 Python 文档导入 Sphinx，在 the Python SVN repository 可以找到一个转换器。它可以较为通用地把 Python doc 风格的的 LaTeX 标记转换为 Sphinx 的 reST。
  • Marcin Wojdyr has written a script to convert Docbook to reST with Sphinx markup; it is at GitHub.
  • Christophe de Vienne 写了一个工具，可以把 Open/LibreOffice 的文档转换到 Sphinx：odt2sphinx。
  • 如果要转换不同的标记语言，Pandoc 可以帮到你。

系统需求¶

Sphinx 至少需要 Python2.7 或者 Python 3.4 才可运行，另外也需要 docutils 和 Jinja2 库。Sphinx 可以配合 docutils 0.10 版本或者其他一些（没有错误的）SVN 主干。如果你还想要源代码高亮的支持，那么也要安装 Pygments 库。
