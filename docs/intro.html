简介¶

这里是 Sphinx 文档生成器的官方文档。Sphinx 软件可以把一组 reStructuredText 格式的源文件转换成多种不同的输出格式，同时自动生成交叉引用、索引等等。就是说，如果你有一个目录里有很多 reST 格式的文档（而且可能子目录里也有），Sphinx 可以（在另一个目录）生成漂亮的结构化的 HTML 文件，以便浏览和导航。不仅如此，使用同样的源文件，还可以使用 LaTex，rinohtype 或 rst2pdf 等软件生成 PDF 格式的文件（详见 可用的构建器）。

本项目主要处理手工录入的文档，而不是自动生成的 API 文档。而且其实 API 文档也是支持的（甚至可以方便地插入到手工录入的文档内容之间），如果你只需要生成单纯的 API 文档，可以看看 Epydoc，它也能理解 reST 语法。

如果需要查阅通用的文档写作方面的知识，解答心里的“为什么”和“怎么做”，可以参考 Write the docs，作者是 Eric Holscher。

从其他文档格式导入¶

这个部分列举转换文档到 reStructuredText/Sphinx 的小技巧。

  • Gerard Flanagan 写了一个脚本可以把 HTML 文件转换为 reST，参见 Python Package Index。
  • 如果要把旧的 Python 文档导入 Sphinx，在 the Python SVN repository 可以找到一个转换器。它可以较为通用地把 Python doc 风格的的 LaTeX 标记转换为 Sphinx 的 reST。
  • Marcin Wojdyr 写了一个脚本，可以把 Docbook 转换为 Sphinx 的 reST 格式，项目地址在 GitHub。
  • Christophe de Vienne 写了一个工具，可以把 Open/LibreOffice 的文档转换到 Sphinx：odt2sphinx。
  • 如果要转换不同的标记语言，Pandoc 可以帮到你。

系统需求¶

Sphinx 至少需要 Python2.7 或者 Python 3.4 才可运行，另外也需要 docutils 和 Jinja2 库。Sphinx 可以配合 docutils 0.10 版本或者其他一些（没有错误的）SVN 主干。如果你还想要源代码高亮的支持，那么也要安装 Pygments 库。
