sphinx-build¶

Synopsis¶

sphinx-build [options] <sourcedir> <outputdir> [filenames …]

Description¶

sphinx-build generates documentation from the files in <sourcedir> and places it in the <outputdir>.

sphinx-build looks for <sourcedir>/conf.py for the configuration settings. sphinx-quickstart(1) may be used to generate template files, including conf.py.

sphinx-build can create documentation in different formats. A format is selected by specifying the builder name on the command line; it defaults to HTML. Builders can also perform other tasks related to documentation processing.

By default, everything that is outdated is built. Output only for selected files can be built by specifying individual filenames.

For a list of available options, refer to sphinx-build -b.

Options¶

-b buildername¶

这是最重要的参数：选择构建器。常用的构建器有：

html
构建 HTML 页面。这是默认的构建器。
dirhtml
构建 HTML 页面，每个文档生成一个单独的文件夹。用于网页服务器，可以显示出好看的 URL（可以不显示.html）。
singlehtml
整个文档构建为单个 HTML 文件。
htmlhelp, qthelp, devhelp, epub
构建 HTML 文件，并附加一些信息，然后输出这些格式的文件。
applehelp
构建苹果帮助手册（Apple Help Book）。需要 hiutil 和 codesign 程序，它们不是开源软件，而且只在 Mac OS X 10.6 以上版本中提供。
latex
构建 LaTeX 源文件，以便使用 pdflatex 编译为 PDF 文档。
man
为 UNIX 系统构建 groff 格式的手册（man pages）。
texinfo
构建 Texinfo 文件，以便使用 makeinfo 程序处理为 Info 文件。
text
构建纯文本文件。
gettext
构建 gettext 样式的文本集（“.pot”格式）。
doctest
如果启动了 doctest 扩展程序，可运行文档中的所有 doctest 测试。
linkcheck
检查所有外部链接的资源是否可以访问。
xml
构建 Docutils-native 的 XML 文件。
pseudoxml
构建紧凑而漂亮的“伪 XML”格式文件，可供审阅文档中的层次结构。

有关 Sphinx 提供的完整构建器，请见 可用的构建器。扩展程序也可以添加一些第三方的构建器。

-M buildername¶

Alternative to -b. Uses the Sphinx make_mode module, which provides the same build functionality as a default Makefile or Make.bat. In addition to all Sphinx builders, the following build pipelines are available:

latexpdf
Build LaTeX files and run them through pdflatex, or as per latex_engine setting. If language is set to 'ja', will use automatically the platex/dvipdfmx latex to PDF pipeline.
info
Build Texinfo files and run them through makeinfo.

重要

Sphinx only recognizes the -M option if it is placed first.

1.2.1 新版功能.

-a¶

If given, always write all output files. The default is to only write output files for new and changed source files. (This may not apply to all builders.)

-E¶

不使用已保存的 :term:`运行环境`（其中缓存了交叉引用列表），每次都重新构建它。默认是只读取和处理新的或者上次运行之后有改动的文件。

-t tag¶

定义指定的 tag 标签。仅供 only 导语使用，只包含它里面的内容。

0.6 新版功能.

-d path¶

因为 Sphinx 需要先读取和处理所有的源文件，然后才能开始输出和写入文件，处理后的源文件会缓存成“doctree pickles”。通常来说，这些文件会放在构建目录中的 .doctrees 文件夹中；使用这个选项你可以选择一个不同的缓存文件夹（不同的构建器可以共用同一个缓存文件夹）。

-j N¶

Distribute the build over N processes in parallel, to make building on multiprocessor machines more effective. Note that not all parts and not all builders of Sphinx can be parallelized. If auto argument is given, Sphinx uses the number of CPUs as N.

1.2 新版功能: 这是一个“实验性”功能。

在 1.7 版更改: Support auto argument.

-c path¶

不使用默认的文档根目录存放配置文件 conf.py，指定另外的配置文件路径。注意可能会有一些文件或者路径是相对于配置文件目录的，所以记得要把这些文件也放在这个指定的位置。

0.3 新版功能.

-C¶

不使用配置文件，只使用 -D 选项提供的参数。

0.5 新版功能.

-D setting=value¶

使用指定的值替换掉配置文件 conf.py 中的值。这里设置的值的类型只能是数字、字符串、列表或者字典。

列表类型，使用逗号分隔：-D html_theme_path=path1,path2。

字典类型，指定变量名和键，再写值，例如：-D latex_elements.docclass=scrartcl。

布尔型（是、否），使用 0 或者 1。

在 0.6 版更改: 现在可以使用字典类型了。

在 1.3 版更改: 现在也可以使用列表类型了。

-A name=value¶

在 HTML 模板中，为变量 name 指定值 value。

0.5 新版功能.

-n¶

使用 nit-picky 模式。目前来说，就是对每个缺少的引用给出 warning。使用配置 nitpick_ignore，可以把某些引用标为“已知缺失”。

-N¶

不使用彩色输出。

-v¶

增加 verbose 程度（loglevel）。此选项可以输出多达三倍的日志。此选项会打开 -T 选项。

1.2 新版功能.

-q¶

标准输出不输出任何内容，只把 warning 和 error 写到标准错误输出。

-Q¶

标准输出不输出任何内容，也不输出 warning。只把 error 写到标准错误输出。

-w file¶

把 warning 和 error 写到指定文件，标准错误输出也会输出。

-W¶

把 warning 变成 error。这意味着遇到第一个 warning 时，构建过程就会停下，而且 sphinx-build 退出时会返回状态 1。

-T¶

遇到未处理的异常时，显示完整的跟踪记录（traceback）。否则只显示一个简要的说明，详细的跟踪记录保存为文件以供后续分析。

1.2 新版功能.

-P¶

（仅供调试使用。）如果构建过程中遇到未处理的异常，运行 Python 调试器 pdb。

-h, --help, --version¶

显示使用方法简介，或者 Sphinx 版本号。

1.2 新版功能.

You can also give one or more filenames on the command line after the source and build directories. Sphinx will then try to build only these output files (and their dependencies).

Environment Variables¶

The sphinx-build refers following environment variables:

MAKE

A path to make command. A command name is also allowed. sphinx-build uses it to invoke sub-build process on make-mode.

Makefile Options

由 sphinx-quickstart 创建的 Makefile 和 make.bat 文件，调用 sphinx-build 时，通常只有 -b 和 -d 两个选项。不过它们也支持如下变量来定制他们的操作：

PAPER

The value for ‘“papersize”` key of latex_elements.

SPHINXBUILD

这个变量提供一个程序名，代替 sphinx-build。

BUILDDIR

指定构建目录，而不使用 sphinx-quickstart 所指定的。

SPHINXOPTS

其他传递给 sphinx-build 的选项。

已弃用告警¶

如果某个 Sphinx 扩展程序使用了已经弃用的功能，会出现已弃用告警，类似于 RemovedInSphinxXXXWarning。如果看到这种告警，请把它报告给扩展程序的作者。

要关闭已弃用告警请给指定的运行环境设置 PYTHONWARNINGS= 环境变量。例如：

  • 运行 PYTHONWARNINGS= make html （适用于Linux/Mac系统），注意中间的空格
  • 先运行 export PYTHONWARNINGS= 然后 make html （适用于Linux/Mac系统）
  • 先运行 set PYTHONWARNINGS= 然后 make html （适用于 Windows 系统）
  • 修改你的 Makefilie 和/或 make.bat 文件，设置其中的环境变量
