sphinx-quickstart¶

语法¶

sphinx-quickstart

说明¶

sphinx-quickstart 是一个交互式命令行工具，它会问你一些问题，根据你对项目的描述，创建一个完整的文档目录结构，以及一个例子 Makefile 可以配合 sphinx-build(1) 使用。

选项¶

-q, --quiet¶

静默模式，跳过交互式提问项目。这个参数需要提供 -p，-a 和 -v 参数。

-h, --help, --version¶

显示使用方法简介，或者 Sphinx 版本号。

目录结构选项

--sep¶

如果设置此选项，将使用不同的源目录与构建目录。

--dot=DOT¶

这个选项在根目录中创建两个文件夹：“_templates”用于自定义 HTML 模板文件，“_static”用于自定义样式表及其他静态文件。你可以指定不同的前缀（比如“.”）代替下划线。

基本配置选项

-p PROJECT, --project=PROJECT¶

项目名称在此设置。（参见 project）。

-a AUTHOR, --author=AUTHOR¶

作者名字。（参见 copyright）。

-v VERSION¶

项目版本。（参见 version）。

-r RELEASE, --release=RELEASE¶

项目发布（号）。（参见 release）。

-l LANGUAGE, --language=LANGUAGE¶

文档语言。（参见 language）。

--suffix=SUFFIX¶

源文件后缀。（参见 :confval:`source_suffix）。

--master=MASTER¶

主文档文件名称。（参见 master_doc）。

--epub¶

使用 epub 格式。

其他扩展选项

--ext-autodoc¶

使用 sphinx.ext.autodoc 扩展程序。

--ext-doctest¶

使用 sphinx.ext.doctest 扩展程序。

--ext-intersphinx¶

使用 sphinx.ext.intersphinx 扩展程序。

--ext-todo¶

使用 sphinx.ext.todo 扩展程序。

--ext-coverage¶

使用 sphinx.ext.coverage 扩展程序。

--ext-imgmath¶

使用 sphinx.ext.imgmath 扩展程序。

--ext-mathjax¶

使用 sphinx.ext.mathjax 扩展程序。

--ext-ifconfig¶

使用 sphinx.ext.ifconfig 扩展程序。

--ext-viewcode¶

使用 sphinx.ext.viewcode 扩展程序。

--extensions=EXTENSIONS¶

使用指定的扩展程序。

Makefile 和 Batchfile 的创建选项

--use-make-mode, --no-use-make-mode¶

Makefile/make.bat 使用（或者不使用）:ref:make-mode<make_mode>。默认值是 使用，这会生成一个更加精确的 Makefile/make.bat 文件。

在 1.5 版更改: 默认为使用 make-mode。

--makefile, --no-makefile¶

创建（或者不创建）makefile。

--batchfile, --no-batchfile¶

创建（或者不创建）批处理文件

项目模板

1.5 新版功能: 用于 sphinx-quickstart 程序的项目模板参数

-t, --templatedir=TEMPLATEDIR¶

指定项目模板文件所在的文件夹。你可以修改 quickstart 生成的 Sphinx 项目文件中的模板文件。以下 Jinja2 模板可供使用：

  • master_doc.rst_t
  • conf.py_t
  • Makefile_t
  • Makefile.new_t
  • make.bat_t
  • make.bat.new_t

详细内容参见 Sphinx 系统提供的模板文件夹。（sphinx/templates/quickstart）

-d NAME=VALUE¶

定义模板变量

参见¶

sphinx-build(1)
